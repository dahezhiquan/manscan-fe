# AGENTS

本仓库的前端开发、联调和接口变更，请遵守以下约定：

## 前端开发规范

- 前端开发必须严格遵循 [`CODE_STYLE.md`](./CODE_STYLE.md) 的全部规范。
- 任何页面、组件、样式、交互、布局或前端重构，在编码前都要先对照 `CODE_STYLE.md`，避免与现有约定冲突。
- 如果 `CODE_STYLE.md` 与当前实现存在差异，优先按文档执行；如确需调整规范，应先同步更新文档。

## 接口联动规范

- 每次前端新增、修改、删除接口调用时，都必须先参考 [`SERVER_API.md`](./SERVER_API.md)。
- 前端实现必须与后端接口定义保持一致，包括请求方法、路径、参数、返回字段、错误码和特殊状态。
- 如果后端接口发生变更，前端接口与调用逻辑必须同步更新，不允许继续沿用旧字段、旧路径或旧约定。
- 每次接口开发或接口变更后，都要补充或更新 [`FE_API.md`](./FE_API.md) 的前端接口文档。
- `FE_API.md` 需要记录前端实际使用的接口信息，至少包含接口用途、请求方式、路径、参数说明、返回结构、异常分支和联调注意事项。

## 后端排查约定

- 当需要判断问题是否来自后端实现时，可以直接查看后端仓库代码，路径为 `/Users/dahe/Documents/DAST/manscan`。
- 查看后端代码仅用于辅助定位问题、核对接口实现和联调行为，不得直接修改后端仓库中的任何代码或文件。

## 变更要求

- 接口相关改动提交前，必须检查前端代码、`SERVER_API.md` 和 `FE_API.md` 三者是否一致。
- 若新增接口文档条目，应确保后续前端实现、联调和测试都能直接依据文档完成。
